BFS Breadth-First Search

BFS بتستكشف الجراف مستوى مستوى (level by level). بتبدأ من Node first وبتزور كل الchild الأول قبل ما تنزل للمستوى التاني. بتستخدم Queue (FIFO) — اللي بيدخل أول بيطلع أول.

Core Data Structures

# الأدوات الأساسية
from collections import deque

queue = deque([start_node])   # FIFO Queue — الطابور
visited = {start_node}        # set — عشان نتجنب التكرار
parents = {start_node: None}  # dict — عشان نعرف المسار

Full BFS Implementation 

from collections import deque

def bfs(graph, start_node, goal_node):
    queue = deque([start_node])         # 1. Init queue with start
    visited = {start_node}              # 2. Mark start as visited
    parents = {start_node: None}        # 3. Track parents for path

    while queue:
        current_node = queue.popleft()  # FIFO: remove from front

        if current_node == goal_node:    # Goal check
            break

        for child in graph[current_node]:
            if child not in visited:
                visited.add(child)
                queue.append(child)
                parents[child] = current_node  # Store parent

    # Path reconstruction (trace back)
    if goal_node in parents:
        path = []
        while goal_node:
            path.append(goal_node)
            goal_node = parents[goal_node]  # Go to parent
        path.reverse()                          # Reverse to get start→goal
        print(path)
Order of operations: Check not in visited BEFORE adding to queue, then visited.add(), then queue.append(). If you check AFTER popping — you may visit same node multiple times!
ترتيب العمليات مهم جداً: بنعمل visited.add() لما بنضيف للـ queue مش لما بنـ pop. لو بنعمله لما بنـ pop، ممكن نضيف نفس العقدة للـ queue أكتر من مرة!

Key Properties (حاسس اننا دخلنا هنا في الداتااستراكشر شوية)

  • Complete: Yes — will always find solution if one exists
  • Optimal: Yes — finds shortest path (in unweighted graphs)
  • Time Complexity: O(V + E)
  • Space Complexity: O(V) — stores all nodes in queue
  • Data Structure: Queue (deque) — popleft() = FIFO

DFS Depth-First Search

DFS بتنزل عمق في الجراف قدر ما تقدر قبل ما ترجع. بتستخدم Stack (LIFO) — اللي بيدخل آخر بيطلع أول. الفكرة: خش في أعمق طريق ممكن، لو وصلت لـ dead end ارجع وجرب طريق تاني.

Full DFS Implementation (from lecture)

def dfs(graph, start, goal):
    stack = [start]       # Stack (list) — LIFO
    pathList = [start]   # Track visited order

    while stack:
        node = stack.pop()  # LIFO: remove from back

        if node == goal:
            return pathList  # Goal found

        for neighbor in reversed(graph.get(node, [])):
            stack.append(neighbor)
        if(neighbor not in pathList):
            pathList.append(neighbor)

    return None  # No path found
Why reversed()? When pushing to a stack, the first child should be processed first. Since stack pops from the end, we push in reversed order so the first child ends up on top.
Example: children = [B, C] → push reversed = [C, B] → stack top = B → B pops first ✓
ليه بنستخدم reversed()؟ لأن الـ stack بيطلع من الآخر، لو عندنا أولاد [B, C] وعايزين B يتزار الأول، لازم نـ push بترتيب معكوس [C, B] عشان B يبقى فوق.

DFS Properties

  • Complete: Not guaranteed (infinite graphs / cycles)
  • Optimal: No — may not find shortest path
  • Time Complexity: O(V + E)
  • Space Complexity: O(depth) — uses less memory than BFS
  • Data Structure: Stack (list) — pop() = LIFO

Critical Syntax Tricks

popleft() vs pop()

# BFS uses deque — popleft() removes from FRONT
queue.popleft()   # O(1) — efficient!

# DFS uses list — pop() removes from END (LIFO)
stack.pop()       # O(1)

# WRONG for BFS: list.pop(0) is O(n) — slow!
# DO NOT use list.pop(0) for BFS

Graph Representation

# Adjacency list (dict of lists) — used in lecture
graph = {
    'A': ['B1', 'C2'],     # A connects to B1 and C2
    'B1': ['D1', 'E1'],
    'C2': ['D2', 'G'],
    'G': []                   # Goal — no children
}

visited Set vs. List

# Use SET for visited — O(1) lookup
visited = set()
visited.add('A')
'A' in visited   # True — O(1)

# Using list would be O(n) for 'in' check — inefficient

Path Reconstruction Pattern

# parents dict maps child → parent
parents = {'A': None, 'B': 'A', 'G': 'B'}

# Trace back from goal to start
path = []
node = 'G'
while node:
    path.append(node)
    node = parents[node]
path.reverse()  # ['A', 'B', 'G']
The while loop stops when parents[node] returns None (start node). Since None is falsy in Python, while node: stops when node becomes None.
